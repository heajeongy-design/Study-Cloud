# Azure 개념 정리 (Q&A)

## 1) Azure VM: 중지(Stopped) vs 할당 해제(Deallocated) 차이점
**핵심 차이**
- **중지(Stopped)**: OS만 종료되고 **CPU/메모리 등 컴퓨팅 리소스는 유지**됨  
  → **재시작이 빠름**, 하지만 **컴퓨팅 요금이 계속 청구될 수 있음**
- **할당 해제(Deallocated)**: **컴퓨팅 리소스(CPU 등)를 반납**함  
  → **컴퓨팅 요금이 부과되지 않음**(일반적으로), 대신 다시 켤 때 리소스 재할당 과정이 필요

---

## 2) 고가용성 구성: 장애 도메인(Fault Domain) & 업데이트 도메인(Update Domain) 분산 목적
**목적**
- 물리적 서버/랙 장애(전원, 네트워크, 호스트 문제)나
- 호스트 유지보수/업데이트(재부팅 등)

가 발생해도 **모든 VM이 동시에 영향을 받지 않게 분산**하여  
-> **서비스 중단을 최소화**하기 위함

---

## 3) Azure Virtual Network(VNet)의 가장 기본적 역할
- Azure 클라우드 내 리소스(VM, DB 등)가 **서로 안전하게 통신할 수 있는 프라이빗 네트워크** 제공
- 네트워크 격리, 서브넷 구성, 보안 정책(NSG 등) 적용의 기반

---

## 4) Azure Storage 사용의 전제 조건이자 최상위 관리 요소
**정답: Storage Account(스토리지 계정)**  
- Blob / File / Queue / Table 등 **모든 스토리지 객체를 담고 관리하는 최상위 컨테이너**

---

## 5) Azure에서 VM vs Container의 가장 중요한 개념적 차이
**VM (Virtual Machine)**
- 하이퍼바이저 위에서 **서버 전체 가상화**
- **게스트 OS 포함**(OS + 앱 단위)
- 격리가 강하지만 상대적으로 무거움

**Container**
- **호스트 OS 커널을 공유**하는 **애플리케이션 레벨 가상화**
- 실행에 필요한 **코드/라이브러리만 패키징**
- 가볍고 빠르며 이식성이 좋음
