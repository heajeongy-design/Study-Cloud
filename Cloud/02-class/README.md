# 02. Azure 지역(Region)과 가용성(Availability), 리소스 관리


## 1) 지리/지역 개념


### 지리적 위치(Geographies)
- 전 세계에 위치한 **데이터 센터 범위/권역**

### 지역(Region)
- 하나 이상의 데이터 센터로 구성된 **데이터 센터 집합체**

### 지역 페어(Region Pair)
- 특정 Region이 **다른 Region과 쌍(pair)** 으로 연결되는 구조
- ⚠️ 모든 Region에서 **모든 서비스/인스턴스 크기**를 사용할 수 있는 것은 아님  
  → **배포 전에 Region별 지원 여부를 반드시 확인**

**참고 링크(Region 선택 가이드)**  
- Choose the Right Azure Region for You | Microsoft Azure

## 2) 가용성(Availability) 개념

### 가용성(Availability) 정의
- 어떤 요소/서비스에 문제가 생겼을 때, **대체 요소가 동일한 서비스/기능을 계속 제공**할 수 있도록 하는 능력

### 가용성 옵션 비교(요약)
- **단일 VM**: 손쉬운 Lift & Shift (가용성 보호 수준 낮음)
- **가용성 집합(Availability Set)**: **데이터 센터 내** 장애(하드웨어/유지관리)로부터 보호
- **가용성 영역(AZ, Availability Zone)**: **전체 데이터 센터 단위** 장애로부터 보호
- **지역 쌍(Region Pair)**: **데이터 보존 경계 내**에서 지역(Region) 수준 보호


---


## 3) 가용성 집합(Availability Set)

### 목적
- 유지관리 또는 하드웨어 오류 발생 시에도 **애플리케이션을 온라인 상태로 유지**하는 구성

### 업데이트 도메인(UD, Update Domain)
- 예약된 유지관리/성능/보안 업데이트가 **순차적으로 적용되는 그룹**
- 예시 흐름: UD0 → UD1 → UD2 순으로 업데이트 진행 (한 번에 전부 내려가지 않게 분산)

### 오류 도메인(FD, Fault Domain)
- 데이터 센터의 **하드웨어(랙, 전원, 네트워크 등) 단위 이중화**를 통해
  장애 발생 시 워크로드를 **물리적으로 분리**하는 개념
- 예: FD0의 랙에 문제가 생겨도, 동일한 워크로드가 FD1/FD2에 분산되어 영향 최소화


---


## 4) 가용성 영역(AZ, Availability Zone)

- Azure Region 내에서 **물리적으로 분리된 위치**
- 가용성 집합보다 **한 단계 높은 보호 수준**
- 각 영역은 **전력/냉각/네트워킹이 독립적**인 하나 이상의 데이터 센터로 구성
- 한 가용성 영역이 중단되더라도 **다른 영역은 계속 동작**
- AZ를 지원하는 서비스는 **독립적인 글로벌 서비스 구성**이 가능


---


## 5) 리소스 그룹(Resource Group)

- 수명 주기를 공유하는 여러 리소스를 묶는 **컨테이너**
- 예: Web + DB + VM + Storage 등을 **한 리소스 그룹**에 포함 가능
- 리소스는 원칙적으로 **하나의 리소스 그룹에만 속해야 함**
- 리소스 수준에서 **RBAC(역할 기반 액세스 제어)**로 보호 가능


---


## 6) Azure Resource Manager(ARM)

- Azure 구독 리소스를 생성/업데이트/삭제할 수 있는 **계층적 관리 구조**
- 주요 기능:
  - 리소스 및 리소스 그룹 생성/구성/관리/삭제
  - 리소스 구성 및 액세스 제어
  - 다양한 도구 및 SDK(SDK: Software Development Kit)를 통한 **관리 자동화**
